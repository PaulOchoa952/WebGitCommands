Comandos de Git

Comando Parametros Descripción Ejemplo
git config --global user.name "[nombre-apellido]" --global indica que deseas configurar a nivel global
usern.name:En este caso es tu nombre de usuario
[nombre-apellido]": Aquí debes reemplazar con tu nombre y apellido reales. 		Establecer un nombre que sea identificable para cuando se revise el historial de versiones git init
git config --global user.email "[email válido]" --global indica que deseas configurar a nivel global
usern.email:En este caso es tu email
[email]": Aquí debes reemplazar con tu email valido. 		Establecer una dirección de correo electrónico que se asociará a cada commit del historial git config --global user.email "paalochoame@ittepic.edu"
git config --global -e --global:Configurando una opción de Git a nivel global en tu sistema. -e: Abre la configuración global de Git en un editor de texto. Fichero con la información de configuración de Git git config --global -e
Inicializacion
git init El directorio se convierte en un repositorio Git local y se crea un subdirectorio oculto llamado ".git" Inicializar un directorio existente como repositorio Git git init
git clone [url] [url]:url del repositorio existente Recuperar un repositorio completo desde una ubicación alojada a través de una URL git clone https://github.com/PaulOchoa952/WebGitCommands.git
Escenario
git status Estatus del repositorio Mostrar los archivos modificados en el directorio de trabajo, preparados para su próximo commit git status
git add [fichero] Puedes reemplazar [fichero] con el nombre del archivo que deseas agregar al área de preparación Añadir un archivo para el próximo commit (escenario) git add archivo.txt
git add -u -u: Es una bandera que significa "update" (actualizar). Actualiza el índice sólo donde ya tiene una entrada que coincide con git add -u
git reset [fichero] Debes reemplazar [fichero] con el nombre del archivo que deseas restablecer o revertir a la última versión confirmada. Desenvolver un archivo conservando los cambios en el directorio de trabajo git reset archivo.txt
git reset −− soft HEADˆ --soft: Esta bandera indica que deseas realizar un reset "suave"
HEAD^ se refiere al commit anterior al último. 		Desenvolver el último commit conservando los cambios en el directorio local git diff --staged
git diff −−staged --staged: Esta bandera indica que deseas ver las diferencias entre los archivos que están en el área de preparación (staging area) y la última confirmación (commit). Diferencia de lo que se ha puesto en escena pero aún no se ha hecho commit git diff --staged
git commit -m “[mensaje descriptivo]” [mensaje descriptivo]:Esta sera la descripción del commit Hacer un nuevo commit con su respecta descripción git commit -m "Agregando nuevos cambios al proyecto"
Ramas
git branch Listar las ramas y aparecerá un * al lado de la rama actualmente activa git branch main,master
git branch [nombre de la rama] [nombre de la rama]:aqui pondras el nombre que llevara tu nueva rama Crear una nueva rama en el commit actual git branch
feature/nueva-funcionalidad
git checkout -b [nombre de la rama] [nombre de la rama]:aqui pondras el nombre de la rama Creación y desplazamiento a la rama indicada git checkout master
git branch -d [nombre de la rama] [nombre de la rama]:aqui pondras el nombre de la rama Elimina la rama indicada git branch -d master
git checkout Cambiar a otra rama y comprobarla en el directorio de trabajo git checkout rama2
git checkout −− . --: Esta doble barra guion (--) se utiliza para separar las opciones de los comandos
.: El punto (.) representa el directorio actual. 		Retomar el estado del último commit git checkout -- .
git merge [rama] [nombre de la rama]:aqui pondras el nombre de la rama Fusionar el historial de la rama especificada en la actual git merge ochoaramos
Inspeccionar y comparar
git log Mostrar todos los commits en el historial de la rama actua git log
Seguimiento de los cambios
git rm [fichero] [fichero]:Aqui va el nombre del fichero que deseas modificar Eliminar el archivo del proyecto y preparar la eliminación para el commit git rm mi-archivo.txt
Ignorando los patrones
git config –global core.excludesfile [fichero] --global: Esta bandera indica que deseas configurar la opción a nivel global
core.excludesfile:Especifica la ubicacion del archivo.
[fichero]:ruta completa del archivo a remplazar 		Git ignora ese fichero para todos los repositorios git config --global core.excludesfile ~/.gitignore_global
Compartir y actualizar
git remote add [alias] [url] [alias]: Debes reemplazar [alias] con un nombre corto o alias
[url]: Debes reemplazar [url] con la URL del repositorio remoto al que deseas conectarte		 Añadir una URL de git como alias git remote add origin https://github.com/tu-usuario/tu-repositorio.git
git fetch [alias] [alias]: Debes reemplazar [alias] con un nombre corto o alias Obtener todas las ramas de ese Git remoto git fetch origin
git merge [alias]/[rama] [alias]: Debes reemplazar [alias] con un nombre corto o alias
[rama]:Nombre de la rama deseada 		Fusionar una rama remota en su rama actual para actualizarla git merge origin/main
git push [alias] [rama] [alias]: Debes reemplazar [alias] con un nombre corto o alias
[rama]:Nombre de la rama deseada 		Transmitir los commits de la rama local a la rama del repositorio remoto git push origin main
git pull Obtener y fusionar los commits de la remota a la rama local git pull origin main
Rescribir la historia
git rebase [rama] [rama]:Nombre de la rama especificada Aplicar cualquier commit de la rama actual por delante de la especificada git checkout mi-rama-feature
git rebase main
git reset −−hard [commit] --hard: Esta bandera indica que deseas realizar un reset "duro"
[commit]: Debes reemplazar [commit] con el identificador (hash) del commit al que deseas retroceder		 Despejar el escenario y reescribir el árbol de trabajo a partir del commit específicado git reset --hard abc123

Conclusión

En resumen, estos son algunos de los comandos esenciales de Git que te ayudarán en tu flujo de trabajo de control de versiones.
Es importante comprender cómo y cuándo usarlos para mantener un historial de versiones limpio y eficiente en tus proyectos.
